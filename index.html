所以为什么要用齐次坐标呢?

发表于 2017-03-20

出于某种原因,我们在这里使用四维矩阵

最早看到这个奇怪的论调是在openGL的教程里.第一次学的时候急着让我的箱子转起来,没考虑那么多.后来系统的复习图形学,开始探究这么问题.

也就是说为啥不这样搞,如下图

首先现代的图形sdk都是可编程管线的,你是怎么变化的是自己写进shader里面的.

说法有很多,我大概回忆下各种说法.

  1. 矩阵描述的是运动,这里的四维矩阵不是用来描述三维空间坐标和向量的,是描述运动的.
  2. 是仿射空间不是三维空间
  3. 如果变换矩阵使用三维的,就无法描述带有平移的运动.
  4. 同上,如果要用三维矩阵描述平移,需要ax后面加个b,ax+b
  5. 如果用三维矩阵表示三维空间的运动,那说到底都是在以原点为中心绕圈圈啊,我感觉这个说法对于我这样的菜鸟最好理解ax+by+c=0与ax+by+c+d=0的区别啊.一目了然.

都对好吧,没毛病,老铁.

完备知识体系之神Milo Yip认为:是的是的,这个称号是我封的

仿射仿射空间确实可以通过ax+b来实现,但是对于透视投影会用到齐次坐标.

这些话我都是用自己的话又翻译的..这样方便一点.我是这么理解他的话的,所谓的透视投影对于变换矩阵的要求简单来讲就是需要平移和缩放,就像一个更大或者更小的相似三角形.

归纳一下:
如果用四维矩阵,也就是三维空间的齐次坐标来表示变化.就可以让缩放,平移,旋转,透视等变换在一个矩阵里实现(矩阵乘法结合律).

openGL的方式,从右往左,向量在最右边.然后ax得到结果.在三维空间中,描述一个点的话w分量一般约定为1,描述一个向量的一般约定为0.毕竟1-1=0.没毛病吧.

计算机中字,字长,字节与位

发表于 2016-09-27

抛开机器谈字长,都是耍流氓

  1. 比特==位==bit

    计算机存储的最小单位

    bit就是一个二进制位.

  2. 字节==拜特==Byte

    计算机中以字节为单位存储和解释信息

    1Byte == 8bit 我在8bit后面加不加s的问题上犹豫了五分钟🙄

    一个字节可以存储一个ASCII码的内容

  3. 字==word

    计算机进行数据处理时,一次存取、加工和传送的数据长度称为字(word)。

    一个64位处理器顾名思义一次可以处理64个位,8个字节的数据,所以一个字长就是8

    32位一个word就是4

    c++里一个指针就占用一个机器字长的长度.

  4. 字长

    字长就是计算机字的长度,64位的计算机的字长就是64.

位图索引

发表于 2016-08-17

说一下之前投放系统概述里说到过的位图索引–Bitmap索引.

这一块应该也是业务逻辑为主要内容的工作中最有Trick的部分了.

当一个基本定向的广告(上下线,修改等)由bp告知ws后,ws会转发给位图索引.

位图索引维护N个位图.N为维度个数.(譬如年龄,兴趣,位置,性别等等)

Age位图:

adid_1 adid_2 … adid_20 …
20 0 0 1 1 0
21 0 1 1 1 1
22 1 0 0 1 0
20 0 1 1 0 1
21 1 1 0 1 1
… .. .. .. .. ..
43 0 1 0 0 0
.. .. .. .. .. ..

兴趣位图,位置位图等都是类似的结构.

左边第一列就是这个维度的所有枚举值,这一行剩下的每一个位置都代表了某广告计划是否选择了这个维度的这个枚举值进行投放.

譬如一个广告计划是对北京,20-25岁,女性进行投放,那么就分别对这三个特征维度的位图更新

当一个广告请求从端上传到MS以后,因为请求中带着用户uid,MS会先从redis中取到这个uid的特征,如果用户年龄是22岁,那么在age位图中第一列为22的行中所有值为1的广告计划 (可以用1去做与操作,或者用0做异或,对位图的可以用左移,右移,加减1,与或非和模运算来实现)

然后在location位图中取位置为北京的那一行,同样的在女性中取所有adid,在别的维度取的值为空,然后对取出的三列adid求交集,done!

之后还会有一些别的业务逻辑的规则应用,譬如**有一种广告产品(也是最贵的广告产品),就是定投,广告主通过自己或者第三方挖包,制定一个uid包上传给广告引擎,我们会在redis里维护一个uid--->adid倒排索引.继续和上面的结果求交集.

最后把所有的广告候选集做排序,返回头部(投放概述图中有提到).

感谢强大的C++,感谢内存操作,感谢MTV,CCTV
基本定向从位图索引中拉取,求交只需 1ms!!

我理解中的左值,右值和引用

发表于 2016-08-11

概述

c++11中所有的值分为三种,左值,右值,和将亡值(eXpiring Value)
将亡值的命名是很准确的,也是右值的一种,右值引用就是为了给他 续命.

  • 具名的,可取址的称为左值
  • 表达式,字面常量 "abc",2,true为纯右值
  • 函数返回值,lambda表达式等为将亡值

左值引用是具名变量值的别名，而右值引用则是不具名（匿名）变量的别名。

无论是左值引用还是右值引用,都必须在立即初始化.因为他们并不拥有绑定对象的内存空间只是一个别名.

右值引用

函数ReturnRvalue()返回一个临时变量(将亡值).

下面的b是常用的做法,把临时变量赋值给b,ReturnRvalue()的生命周期在本行被执行后终结.

上面的a是右值引用,他绑定了临时变量,因为临时变量并不会再语句结束后死去,只要a活着,他就不会死.

值得注意的是,如果函数的返回值是一个类的对象,那么当另一个函数直接调用此函数返回值时,如果是值引用会调用拷贝构造和析构函数.

万能引用(常量左值引用)

常量左值引用被称为万能引用,我猜是因为因为他打破了左值引用不可以绑定右值的传说.

常量左值引用是只读的!!!!

目的是为了减少各种开销,内存,复制,拷贝构造,析构等.
